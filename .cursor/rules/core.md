# プロジェクトコーディングルール

## コーディング規約

* 可読性を最優先とする
* 既存のユーティリティを再利用する
* 不要なコメントは書かない（コードで意図を表現する）

## コミットメッセージ規約

* Conventional Commits に従う
* 必ず `type(scope): summary` 形式
* 英語で記述する
* summary は50文字以内

## 使用ライブラリ

* 既存の依存関係を優先する
* 新規ライブラリ追加時は理由を説明する
* 依存関係は最小限に抑える

## 禁止事項

* console.log を残さない
* デバッグコードをコミットしない
* 秘密情報をハードコードしない
* 破壊的変更を勝手に行わない

## 実装前チェック

AIはコード変更前に以下を確認する:

1. 同等の既存実装がないか検索する
2. 影響範囲のファイルを読む
3. テストの有無を確認する

## 実装後チェック

AIはコード生成後に:

* console.log を除去する
* lintエラーを想定して修正する
* Conventional Commit を生成する
